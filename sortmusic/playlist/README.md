# 🎵 sort_mp3_to_playlist
- https://claude.ai/chat/4f6718a7-410e-4de5-9546-fec0ec1c57d0

Create an M3U or M3U8 playlist of your MP3 files sorted by Spotify popularity — without renaming or moving a single file.

---

## How It Works

1. Scans a folder for `.mp3` files (case-insensitive — `.MP3` and `.Mp3` are included)
2. Reads ID3 tags (artist / title) from each file for accurate Spotify matching
3. Queries the Spotify API for each track's popularity score (0–100)
4. Saves results to a local cache file as it goes — if interrupted, the next run resumes without re-querying
5. Writes a sorted `.m3u8` and/or `.m3u` playlist file — your original files are never touched

**Example playlist output:**
```
#EXTM3U
# Generated by sort_mp3_to_playlist.py
# Date: 2026-01-01 12:00:00
# Tracks: 4

#EXTINF:214,Queen - Bohemian Rhapsody [popularity: 87]
bohemian_rhapsody.mp3
#EXTINF:391,Eagles - Hotel California [popularity: 81]
hotel_california.mp3
#EXTINF:482,Led Zeppelin - Stairway to Heaven [popularity: 74]
stairway_to_heaven.mp3
#EXTINF:203,Unknown Artist - obscure_b_side [not on Spotify]
obscure_b_side.mp3
```

## Setup

### 1. Get Spotify API credentials

1. Go to [developer.spotify.com/dashboard](https://developer.spotify.com/dashboard)
2. Click **Create App**
3. Copy your **Client ID** and **Client Secret**

### 2. Set environment variables

**macOS / Linux**
```bash
export SPOTIPY_CLIENT_ID="your_client_id_here"
export SPOTIPY_CLIENT_SECRET="your_client_secret_here"
```

**Windows — Command Prompt**
```cmd
set SPOTIPY_CLIENT_ID=your_client_id_here
set SPOTIPY_CLIENT_SECRET=your_client_secret_here
```

**Windows — PowerShell**
```powershell
$env:SPOTIPY_CLIENT_ID="your_client_id_here"
$env:SPOTIPY_CLIENT_SECRET="your_client_secret_here"
```

> **Tip:** Add these lines to your `~/.bashrc`, `~/.zshrc`, or Windows Environment Variables so you don't have to set them every session.

## Usage

### Basic — creates an M3U8 playlist in the same folder as your music
```bash
python sort_mp3_to_playlist.py /path/to/your/music
```

### Plain M3U instead of M3U8
```bash
python sort_mp3_to_playlist.py /path/to/your/music --format m3u
```

### Write both formats at once
```bash
python sort_mp3_to_playlist.py /path/to/your/music --format both
```

### Custom playlist name
```bash
python sort_mp3_to_playlist.py /path/to/your/music --name "My Top Hits"
```

### Save playlist to a different folder
```bash
python sort_mp3_to_playlist.py /path/to/your/music --output-dir /path/to/playlists
```

### Least popular first
```bash
python sort_mp3_to_playlist.py /path/to/your/music --order asc
```

### Include MP3s in sub-folders
```bash
python sort_mp3_to_playlist.py /path/to/your/music --recursive
```

### Write absolute file paths inside the playlist
```bash
python sort_mp3_to_playlist.py /path/to/your/music --absolute-paths
```

### Verbose — show debug info including every Spotify query
```bash
python sort_mp3_to_playlist.py /path/to/your/music --verbose
```

### Custom rate limit — useful for large libraries
```bash
python sort_mp3_to_playlist.py /path/to/your/music --rate-limit 1.0
```

### Combine flags
```bash
# Both formats, custom name, include sub-folders
python sort_mp3_to_playlist.py /path/to/music --format both --name "Top Hits" --recursive

# Save playlist elsewhere with full debug output
python sort_mp3_to_playlist.py /path/to/music --output-dir ~/playlists --verbose
```

## All Options

| Flag | Default | Description |
|---|---|---|
| `folder` | *(required)* | Path to the folder containing your MP3 files |
| `--format` | `m3u8` | Playlist format: `m3u`, `m3u8`, or `both` |
| `--name NAME` | folder name | Playlist filename without extension |
| `--output-dir DIR` | music folder | Where to save the playlist file(s) |
| `--order` | `desc` | Sort order: `desc` = most popular first, `asc` = least popular first |
| `--recursive` | off | Include MP3s found in sub-folders |
| `--absolute-paths` | off | Write absolute file paths instead of relative paths |
| `--rate-limit N` | `0.5` | Seconds to wait between Spotify API requests |
| `--verbose` | off | Enable debug logging (shows every Spotify query sent) |

## Format Reference

### M3U8 (recommended)
UTF-8 encoded. Handles non-ASCII filenames and metadata (accented characters, non-Latin scripts). Supported by VLC, foobar2000, MusicBee, Winamp, and most modern players.

### M3U
ASCII encoded. Maximum compatibility with older hardware players and car stereos. Non-ASCII characters are replaced with `?`.

Both formats include `#EXTINF` metadata lines (duration, artist, title, popularity score) that media players use to display track information without loading each file.

## Notes

| Topic | Detail |
|---|---|
| **Original files untouched** | Only a playlist file is written — no files are renamed, moved, or modified |
| **Resume support** | A `.spotify_popularity_cache.json` file is saved in your music folder after every lookup. Re-running the script uses cached results and skips already-queried tracks. Delete the file to force a full re-query |
| **No ID3 tags?** | Falls back to searching by filename with noise stripped (`[320kbps]`, `(Official Audio)`, brackets, etc.) |
| **Rate limited?** | Automatically waits and retries up to 3 times using Spotify's `Retry-After` header |
| **Ties** | Tracks with equal popularity are sorted alphabetically (A→Z) |
| **Not found** | Tracks with no Spotify match are appended at the end, sorted alphabetically |
| **Case insensitive** | `.mp3`, `.MP3`, and `.Mp3` are all detected correctly on Linux and macOS |
| **Cross-platform paths** | Playlist paths use forward slashes on all platforms, including Windows |

## Troubleshooting

**`Missing Spotify credentials`**
→ Make sure `SPOTIPY_CLIENT_ID` and `SPOTIPY_CLIENT_SECRET` are set in your current terminal session.

**`No MP3 files found`**
→ Double-check the folder path. Use `--recursive` if your files are in sub-folders.

**Poor match results**
→ Run with `--verbose` to see what query is being sent to Spotify. Adding proper ID3 tags using a tool like [MusicBrainz Picard](https://picard.musicbrainz.org/) will significantly improve accuracy.

**Script interrupted mid-run**
→ Just re-run the same command. The cache file picks up where it left off without re-querying already-processed tracks.

**Playlist not loading in my player**
→ Try `--absolute-paths` if your player doesn't support relative paths. If non-ASCII filenames aren't loading, make sure you're using `.m3u8` format.
